IMU Filtering

Keywords: Applied Mathematics, MATLAB, Kalman Filter

My first main task on UMD Loop is to filter the data that this IMU gives back to obtain an accurate reading for the acceleration of the pod. We'd also like to use the acceleration for an approximate solution for velocity. An approach suggested by the Avionics lead was to use quaternions and rotate the gravity vector iteratively so that we can remove it accurately for all time.


But, as you can see it's a little noisy and error prone...

Currently being looked into:

  1. Thresholding higher-order derivatives of motion
  2. Using the accelerometer to stop recording if not in motion
  3. Curve fitting
  4. Complementary Filter
  5. Kalman Filter

Power Supply GSE Software

Keywords: ROS, QT, GUI, Python, multithreading, TCP/IP, Power Supply Communication

This one was my first ever project in the lab. My task was to develop software to allow the robot operator to communicate with the power supply connected to the robot. This was really my introduction to ROS and QT GUI development. I'm pretty lucky that I was exposed to so much on my first project. The software for this was particularly finicky since it had to be configurable to any number of available power supplies. I ended up taking full advantage of Object-Oriented-Programming by setting up abstractions for the types of power supplies since they all communicated and behaved differently - so perfect for OOP! I even had to make a (sort-of since it was in Python) multi-threaded implementation to have the power supply object publish in ROS while locking over the TCP/IP connection since only one power supply could be TCP'd at a time.
